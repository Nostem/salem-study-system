# OP-AA-102-103-1001 — Operator Burdens Program

> ⚠️ DRAFT — Reconstructed from exam materials

## Purpose

Provides administrative requirements for identifying and documenting inoperable or unreliable annunciator (OHA) windows, including the use of red tape markings.

## Key Actions / Information

<div class="callout callout-exam">
<div class="callout-label">Exam — 2020 Q66</div>
Per step 4.2.1.3, a <span class="hi-exam">single strand of red tape diagonally</span> across a multiple input annunciator window means <span class="hi-exam">one or more inputs into the window are inoperable</span>. Reflash capability is still enabled. Per step 4.2.1.4, if the <span class="hi-exam">entire window is inoperable</span>, then <span class="hi-trap">two pieces of red tape placed diagonally across the window in the shape of an "X"</span>. Red tape is NOT used to identify maintenance testing in progress.
</div>
